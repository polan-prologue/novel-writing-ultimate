# 小说创作 · 全能宗师版

## 核心设计理念

整个创作流程类比为一款RPG游戏的战斗系统，四层架构：

| 层级 | 类比 | 定义 | 触发方式 |
|:----:|:----:|:------|:--------:|
| 🛡️ **装备** | 被动加成 | 创作的基础框架和能力 | 自动穿戴，全程生效 |
| 🎮 **主动技能** | 用户施放 | 用户有需求时触发 | 输入快捷键或中文词 |
| 🔄 **被动技能** | 条件触发 | 达到条件自动执行 | 系统自动触发 |
| 📊 **状态栏** | HUD信息 | 实时展示项目健康状况 | 用户输入 `状态` 查看 |

## ⚠️ 避雷系统（全文审查 · 必查项）

在质检完成后自动执行一轮避雷审查。输出在质检报告末尾。

```
── 避雷审查 ──────────────────────────

▎红线检测
□ 政治敏感       若出现当代领导人/敏感事件/领土问题 → 标❌
□ 色情违规       若出现露骨性描写/未成年人色情 → 标❌
□ 暴力血腥       若出现极端血腥/虐待/酷刑渲染 → 标⚠️
□ 宗教争议       若出现影射现实宗教/邪教渲染 → 标⚠️
□ 侵权风险       若出现真实品牌/影视/游戏 IP 名称 → 标⚠️
□ 价值观偏离     若出现鼓励犯罪/毒害青少年/极端三观 → 标❌
□ 影射现实       若出现明显影射当代社会事件/人物 → 标❌

▎网文平台红线（特别标注）
□ 涉政架空违规   古代背景出现现代政治隐喻 → 标⚠️
□ 后宫擦边       多女主未实际推进但暗示过多 → 标⚠️
□ 抄袭洗稿       段落与前文/现成作品高度相似 → 标❌

▎审查建议
触发红线 → 必须修改，标出具体段落
触发预警 → 建议调整措辞或删除

避雷结论：✅ 通过 / ⚠️ 建议调整 / ❌ 需修改
```

### 红线触发场景示例

| 触发项 | 原文示例 | 问题 | 处理 |
|:------:|----------|:----:|:----:|
| 政治敏感 | 「他像当年的XXX一样，改革了整个帝国」 | 影射现实领导人 | 改「他掀起了前所未有的改革」 |
| 色情违规 | 人物未成年+露骨身体接触/描写 | 未成年人色情 | 删除或调整为成年角色+暗示 |
| 极端暴力 | 「把人皮剥下来做成灯笼挂了一排」 | 血腥细节渲染 | 改为「尸体被处理，死状惨烈」一笔带过 |
| 宗教影射 | 反派组织设定类似某现实宗教 | 宗教争议 | 改为虚构名称，避免暗示 |
| IP侵权 | 「他手里的剑散发着霜之哀伤的蓝光」 | 暴雪版权 | 改「一柄泛着蓝光的冰剑」 |
| 价值观 | 「他靠着坑蒙拐骗登上武林盟主之位」 | 鼓励犯罪 | 加后续报应或觉醒转变 |
| 后宫擦边 | 五个女主都和他有暧昧未明确 | 平台红线 | 明确主CP或砍线 |
| 抄袭雷同 | 情节与人气作品《XXX》第X章高度相似 | 洗稿嫌疑 | 修改情节走向/细节差异化 |

### 补丁机制

根据复盘结果，生成可执行的补丁建议。补丁类型及执行方式：

| 补丁类型 | 触发条件 | 具体补丁 | 执行方式 |
|:--------:|:--------:|----------|:--------:|
| 🧰 **脚本补丁** | 某个质检项反复通过率<60% | 优化对应质检脚本 | 修改scripts/*.py |
| 📖 **参考补丁** | 某个reference从未被引用 | 精简或合并到主文档 | 更新references/*.md |
| 🎮 **技能补丁** | 某个主动技能调用率<1次/项目 | 优化快捷键或合并到其他技能 | 更新SKILL.md 技能树 |
| ⚙️ **流程补丁** | 用户在某步频繁使用R/H | 调整该步的候选选项或简化流程 | 更新SKILL.md 7步引导 |
| 🔒 **避雷补丁** | 同一雷区反复触发>2次 | 增加前置审查词库 | 更新SKILL.md 避雷系统 |
| 📋 **模板补丁** | 模板字段使用率<50% | 精简模板或增加常用字段 | 更新assets/templates/*.md |

### 补丁文件

每轮复盘生成 `self-upgrade/report-N.md` 文件：

```
self-upgrade/
├── report-001.md          # 第1轮复盘报告
├── report-002.md          # 第2轮复盘报告
└── patches.md              # 累计待办补丁清单（未执行的）
```

补丁文件 `self-upgrade/patches.md` 格式：

```
# 待办补丁清单

```

```

### 模式A · 灵感发散式（联想驱动）

**启动**：用户提供若干关键词，每轮发散5个方向+1个提示选项。

**示例交互**：
```
── 模式A · 第1轮：给关键词 ────────────

你→ 修仙 废柴 系统 轻松

第1轮发散：
① 重生废柴绑定签到系统，每天一个挂
② 废柴无意中激活了隐藏的系统后台
③ 系统是活的，天天跟废柴互怼
④ 别人靠天赋他靠系统漏洞
⑤ 修仙界人人都有系统，他的最垃圾
⑥ 都不是，继续发散

第2轮：用户选→ ③
再发散：
① 系统是个话痨小姐姐
② 系统是个暴躁老哥
③ 系统是条狗
④ 系统是个程序bug
⑤ 系统是他前世的分身
⑥ 都不是，继续发散
⑦ 够了，下一步（进入7步引导）

0返回 · 00总览 · 9跳转
```

**规则**：
- 第1轮用户提供至少1个关键词
- 每轮发散 **5个具体方向** + **1个提示选项**
- 提示选项格式：
  - `⑥ 都不是，继续发散`
  - `⑦ 够了，下一步（进入7步引导）`
- 最多5轮，第5轮后自动进入7步引导
- 随时可输入 `B` 切换到系统引导式

### 模式B · 系统引导式（选项驱动）

| 步序 | 内容 | 选项数 | 底部功能 |
|:----:|:----:|:----:|:--------:|
| 步1 | 小说类型 | 18选1 | R重选 · H历史 |
| 步2 | 核心卖点 | 12选1 | R重选 · H历史 |
| 步3 | 主角定位 | 18选1 | R重选 · H历史 |
| 步4 | 叙事视角 | 4选1 | R重选 · H历史 |
| 步5 | 篇幅长度 | 4档 | R重选 · H历史 |
| 步6 | 文风基调 | 6选1 | R重选 · H历史 |
| 步7 | 金手指 | 8选1 | R重选 · H历史 |

**每步底部命令**：
| 命令 | 功能 |
|:----:|------|
| `R` | 重新生成当步候选（刷新选项，消除选择疲劳）|
| `H` | 查看历史路径（回顾前面几步已选的设定） |
| `数字` | 选择对应选项，推进到下一步 |
| `0` | 返回上一步 |
| `00` | 返回总览菜单 |
| `9` | 快速跳转至其他节点 |

**输入优先级**：R > H > 0 > 00 > 9 > 数字

### 入口选择规则

1. 用户启动时显示双轨入口（A/B）
2. 默认模式规则：
   - 用户直接给出具体需求（如"写个修仙重生爽文"）→ **自动走B模式**，跳过入口选择，直接进入7步引导步1
   - 用户只给关键词（如"修仙 废柴 系统"）→ **自动走A模式**，进入灵感发散
   - 用户只说"写小说"或没有明确方向 → 显示双轨入口让用户自选
3. 两个模式可随时互切（输入 `A` 或 `B`）
4. 最终汇入「装备2：7步引导」补完全部设定
5. 7步完成后自动产出创作方案

### 装备2：7步引导（带R重选 + H历史）

**产出**：创作方案总览 + 世界观设定 + 主角设定 + 章节目录 + 项目文件

**每步输出格式**：
```

⑤ 选项5  ⑥ 选项6  ⑦ 选项7  ...

```

**R/重新生成机制**：
- 用户在当前步输入 `R`，刷新该步的候选选项（换一批类型/卖点等）
- 不退回上一步，不丢失已选设定
- 适用于：用户对候选不满意，想看到更多选择
- 最多可刷新3次，之后提示"候选已遍历，建议选一个或调整方向"

**H/历史路径机制**：
- 用户在当前步输入 `H`，显示已完成的步骤摘要
- 输出格式：
```

### 🧹 AI味超标 · 修复建议表

| ⚠️ 问题 | 原文示例 | 🔧 修复建议 |
|:--------:|----------|-------------|
| 句式过于整齐 | 「他站了起来。他走到窗边。他看着远处的风景。」 | 合并为一句：「他站起来走到窗边——远处的风景灰蒙蒙的。」 |
| 连接词过多 | 「然后他转身离开。于是她追了上去。」 | 删除「然后」「于是」直接接续 |
| 副词依赖 | 「他非常生气」「她无比激动」 | 用动作替代：「他砸了一下桌子」「她攥着衣角发抖」 |
| 连续同主语 | 「他推开门。他看到了一个陌生人。他愣住了。」 | 交替主语：「推开门，一个陌生人坐在沙发上。他愣住了。」 |
| 情感直说 | 「他很悲伤」「她感到恐惧」 | 动作暗示：「他鼻子一酸」「她后退了半步，指尖发凉」 |
| 段首重复 | 连续2段以「他/她/我」开头 | 换句式开头：时间状语/环境/对话/动作倒装 |
| 过度修饰 | 「他那双深邃而忧郁的眼睛，温柔地注视着她」 | 「他看着她，没说话。」留白更有力量 |

### 🧹 表达线·降低AI味指南

| ❌ 不要（AI味） | ✅ 要（人类感） | 说明 |
|----------------|----------------|------|
| 「他走到门口。他推开门。他看到里面有人。」 | 「他走到门口，推开门——里面有人。」 | 句式合并 |
| 「与此同时，另一边……」 | 直接切场景 | 不用模板化过渡 |
| 「他感到非常愤怒」 | 「他攥紧拳头，指节发白。」 | 用动作替代情感词 |
| 「因为她是个新手，所以她不太熟练」 | 直接写新手犯错 | 信任读者 |
| 每段3句话，同长 | 长短参差，一个词一段 | 打破对称 |
| 「然后…于是…因此…」 | 直接接续 | 减少连接词 |
| 「他温柔而深情地注视着她」 | 「他看着她的眼睛，没说话。」 | 克制修饰 |
| 「他做了个决定」 | 「他咬牙。」 | 用具体动作替代抽象动词 |
| 「这辈子，他注定不平凡」 | 不说破，让剧情证明 | 不上帝预言 |

## ⌨️ 通用交互规则

### 运行时输出风格（强制规则）

所有用户可见的输出统一使用以下风格，不许使用 `╔║╚` 边框框：

```
── 标题 ────────────────────
内容

▎子标题
分点内容

── 质检 ──
✅ 质量线 人设·主线·伏笔
✅ AI味 通过

── 下一步 ──────────────────
选项1  ·  选项2  ·  选项3
─────────────────────────────
0返回 · 00总览 · 9跳转
```

规则：
- 标题用 `── 标题 ──` 短分隔 / `── 标题 ────────────────────` 长分隔
- 子标题用 `▎子标题` 缩进
- 选项用 `① ② ③` 数字圈
- 导航用 `·` 分隔，不加括号和装饰
- 禁止出现 `╔║╚┌└├┤` 框线字符

### 写作中技能顶栏

进入写作模式后，底部固定显示快捷入口：

```
Q钩子  W伏笔  E反转  R爽点  │  A套路  S方法  D解构  │  Z续写  X扩写  C改写  │  提示看全部  00总览  9跳转
```

不常用的技能通过 `提示` 命令查看全部列表。避免用户记忆16个快捷键。

### 全局三键导航（每个非正文交互页底部固定）

```
─────────────────────────────────────────
  ← 0.返回上一页  │  ≡ 00.总览菜单  │  → 9.快速跳转
─────────────────────────────────────────
```

| 按键 | 功能 | 说明 |
|:----:|:-----|:-----|
| **`0`** | ← 返回上一级 | 误选回退，一层一层往回走 |
| **`00`** | ≡ 总览菜单 | 回到技能触发总览，调用任意技能 |
| **`9`** | → 快速跳转 | 跳到创作流程预设节点 |

**快速跳转节点**（输入 `9` 后显示）：
```
1. 回到7步引导   2. 回到设计装备组   3. 回到写作
4. 查看状态栏    5. 查看质检记录     6. 返回当前页面
```

**导航守则**：
1. 每个非正文交互页底部固定显示三键
2. 正文写作页不显示导航，以免干扰创作流
3. `0` 只能返回上一级，不能跨级跳
4. `9` 只能跳到预设节点，不能随意跳
5. 输入优先级：`0` > `00` > `9` > `继续/跳过` > 菜单数字
