## 标点符号规范

### 基本要求

| 类型 | 要求 |
|------|------|
| 全角/半角 | 中文内容用全角标点，英文用半角 |
| 引号 | 中文内容用「」或“”，配对使用 |
| 书名号 | 《》用于书名、篇名、剧名等 |
| 省略号 | 六个点「……」而非三个点「...」 |
| 破折号 | 两个短横「——」居中 |

### 常见错误

1. **引号不配对**：开引号后无闭引号
2. **书名号嵌套错误**：《书名「篇名」》是正确的嵌套
3. **全角半角混用**：「你好，」而非「Hello，」
4. **连续标点**：「，、」连续使用
5. **标点后多余空格**：「你好。」之后不应有空格

### 脚本检查

执行标点检查：
```bash
python scripts/punctuation_check.py --input ./chapter.txt
```

## 段落结构规范

### 段落长度

| 类型 | 推荐字数 | 说明 |
|------|----------|------|
| 标准段落 | 100-200字 | 主要叙事段落 |
| 短段落 | <100字 | 制造节奏感、强调、对话间隔 |
| 长段落 | >200字 | 仅在特定场景使用，避免滥用 |

### 段落节奏

1. **长短交替**：长段落叙述后接短段落对话
2. **留白适度**：关键情节后可空一行
3. **首句原则**：首句点明本段核心
4. **避免碎句**：不要每句一段

### 段落转换

- 场景切换：空一行或使用「* * *」分隔
- 时间跳跃：简短过渡句说明
- 视角切换：明确标注或分隔

## 一致性检查

### 检查清单

#### 人物设定一致性

- [ ] 外貌特征：身高、体型、五官、发色等前后一致
- [ ] 性格特点：行为反应符合性格设定
- [ ] 能力范围：不出现超出设定的新能力
- [ ] 口头禅/习惯：保持一致

#### 事件逻辑一致性

- [ ] 时间线：事件先后顺序合理
- [ ] 因果关系：事件发展有合理动因
- [ ] 物理逻辑：不违背基本物理规则（如未说明则默认遵循）
- [ ] 物品位置：重要物品不凭空出现或消失

#### 称呼一致性

- [ ] 人名：同一人物称呼固定（姓名/昵称/尊称不混用）
- [ ] 地名：同一地点名称统一
- [ ] 组织名：势力/门派名称固定

#### 代词使用

- [ ] 视角清晰：叙述中代词指代明确
- [ ] 避免歧义：连续多个「他」时适当使用姓名
- [ ] 单复数：注意「他」与「他们」的使用

### 常用一致性检测问题

```
问题1：前面说主角近视，后文却描写他看清远处车牌
问题2：第一章说父亲已故，第五章却出现父亲对话
问题3：配角名字在不同章节有不同写法
```

## 风格一致性

### 叙事视角

| 视角 | 特点 | 一致性要求 |
|------|------|------------|
| 第一人称 | 「我」的所见所感 | 不写其他角色内心 |
| 第三人称全知 | 上帝视角 | 可写任何角色内心 |
| 第三人称限知 | 跟随某角色 | 仅限该角色视角 |

### 叙事语气

- **严肃文学**：用词考究、句式复杂
- **网络小说**：口语化、节奏快、爽点明确
- **悬疑推理**：冷静克制、注重细节
- **言情**：情感细腻、氛围渲染

### 描写密度

| 类型 | 描写比例 | 说明 |
|------|----------|------|
| 高密度 | 描写占50%+ | 沉浸式、氛围感强 |
| 中密度 | 描写30%左右 | 叙事与描写平衡 |
| 低密度 | 描写<20% | 快节奏、情节推进 |

### 用词习惯

- 建立角色专属词汇表
- 避免同一段落重复用词
- 保持时代/地域用词一致

## 降低AI味指南

### AI写作常见特征

| 特征 | 表现 | 改进方法 |
|------|------|----------|
| 机械重复 | 相同句式开头 | 变换句式开头 |
| 过度完美 | 描写过于面面俱到 | 选择性留白 |
| 情感直白 | 「他很悲伤」 | 通过动作暗示 |
| 信息密集 | 一次性输出大量信息 | 分批次、留悬念 |
| 连接词堆砌 | 「首先...其次...然后...最后...」 | 自然过渡 |
| 过度解释 | 「这说明他很紧张」 | 让读者自己理解 |

### 人类化写作技巧

#### 1. 句式变化

```
AI味：「他走进房间。他看到桌子上有一封信。他拿起信。」
人类：「走进房间，一股霉味扑面而来。桌上？什么时候多了封信。」
```

#### 2. 细节选择

```
AI味：「她有一头乌黑亮丽的长发，大大的眼睛，高挺的鼻梁。」
人类：「她甩了甩头发，把遮住眼睛的几缕别到耳后。」
```

#### 3. 情感暗示

```
AI味：「他听到这个消息非常震惊。」
人类：「他张了张嘴，半天没发出声音。」
```

#### 4. 节奏控制

```
AI味：每句都有完整的主谓宾
人类：短句断句、长句穿插、制造呼吸感
```

#### 5. 口语化

```
AI味：「我认为这个方案存在一定的问题。」
人类：「这方案……总觉得哪里不对。」
```

### 自我检查问题

1. 这段话读起来像一个真实的人在说话吗？
2. 是否有「首先、其次、然后」这样的机械连接词？
3. 情感是否通过动作/神态暗示而非直接说出？
4. 细节是否过于完美/全面？
5. 句式是否有变化？

## 逻辑漏洞检测

### 常见逻辑漏洞

#### 时间线漏洞

- 事件A发生在事件B之前，但描述方式暗示相反
- 时间跳跃时未说明，导致读者困惑
- 季节/天气描述与时间线矛盾

#### 因果漏洞

- 事件B的发生缺乏合理的触发原因
- 角色行动没有充分的动机支撑
- 转折过于突兀，缺乏铺垫

#### 能力漏洞

- 角色在困境中使用之前未展现的能力
- 重要道具/信息突然出现无解释
- 配角能力超越设定

#### 细节漏洞

- 之前否定的事实后文又肯定
- 物品在角色不知情的情况下被使用
- 角色具备某些知识但显然不应知道

### 检测方法

1. **倒推法**：从结局倒推每一步是否合理
2. **旁观法**：假设自己是读者，哪里会困惑
3. **清单法**：对照上文提到的细节，检查下文中是否矛盾
4. **大声朗读**：逻辑问题往往朗读时能发现

## 敏感内容处理

### 需要注意的类型

- 政治敏感话题
- 过度暴力/血腥描写
- 色情擦边内容
- 抄袭/融梗风险

### 处理原则

1. 隐喻替代直白
2. 留白而非详写
3. 转换视角/表现方式
4. 确保原创性
